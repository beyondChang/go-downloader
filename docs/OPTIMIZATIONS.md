# Downloader 是如何优化下载的

Downloader 旨在克服标准浏览器下载的局限性，从而最大化下载速度。

## 浏览器下载的问题

标准的浏览器通常只打开一个与服务器的 HTTP 连接。
服务器通常会限制给予单个连接的带宽，以确保对所有用户公平。
下载管理器（如 Downloader）会同时打开大量请求（Downloader 默认为 32 个）。它们利用这些请求将文件分割成许多小块，并分别下载这些小块。

## 连接的差异性

并不是所有的连接都是平等的，由于负载均衡器和 CDN 等原因，有些连接快，有些连接慢。下载管理器有许多方法来优化这些连接。

## Downloader 的核心优化策略

1.  **大分块 (Large chunks):** 我们将下载划分为 `$num_workers`（工作线程数）个部分，并分配给每个工作线程 `$(fileSize/numworkers)` 字节的数据进行下载。这确保了每个连接的数据量足够大且高效。
2.  **健康检查 (HealthCheck):** 我们计算所有工作线程的平均速度。如果有一个工作线程的速度低于平均速度的 0.3 倍，我们会重启它，期望它能获得一条更快到达服务器的新路径。
3.  **窃取工作 (StealWork):** 在下载接近尾声时，当快的工作线程已经完成，而慢的工作线程仍在工作时，我们会让闲置的快线程从慢线程那里“窃取”工作。
4.  **对冲工作 (HedgeWork):** 同样在接近尾声时，如果有闲置的工作线程和活跃的工作线程，并且无法进行窃取工作（分块大小变得太小），我们会让闲置线程执行与活跃线程相同的任务。如果闲置线程先完成，我们将使用它的结果并取消活跃线程的任务。
